HttpProxyRecord

HTTP/HTTPS 트래픽을 캡처·기록·재생하는 크로스플랫폼(macOS + Windows) 데스크톱 앱

최신 버전 다운로드 →

1. 다운로드

GitHub Releases에서 운영체제에 맞는 파일을 받습니다.

  • macOS Apple Silicon / Intel — HttpProxyRecord-*-mac.zip (arm64 / x64)
  • Windows x64 — HttpProxyRecord-Setup-*.exe (NSIS 인스톨러)

2. 설치

macOS

  1. 받은 .zip을 더블클릭해 압축을 풀면 HttpProxyRecord.app이 나옵니다.
  2. 이 .app을 /Applications 폴더로 옮깁니다.
  3. 아래 Gatekeeper 차단 해제를 반드시 수행한 뒤 실행합니다.
⚠️ "손상되었기 때문에 열 수 없습니다" / "확인되지 않은 개발자" 경고가 뜬다면 이 앱은 Apple 코드 서명·공증(notarization)을 받지 않은 무료 앱이라, macOS Gatekeeper가 다운로드한 앱에 격리(quarantine) 속성을 붙여 실행을 막습니다. 터미널에서 아래 한 줄을 실행해 격리 속성을 제거하면 정상 실행됩니다. 
xattr -dr com.apple.quarantine /Applications/HttpProxyRecord.app
앱을 다른 경로에 두었다면 /Applications/HttpProxyRecord.app 부분을 실제 경로로 바꿉니다. 이후 첫 실행 시에는 앱을 우클릭 → 열기로 한 번 실행하면 됩니다.

Windows

  1. HttpProxyRecord-Setup-*.exe를 실행합니다.
  2. SmartScreen 경고가 뜨면 추가 정보 → 실행을 선택합니다 (서명되지 않은 앱).
  3. 설치 위치를 선택하고 설치를 완료합니다.

3. 사용 방법

  1. 녹화 시작 — 세션 이름을 입력하고 시작하면 프록시가 127.0.0.1:8888에 뜹니다.
  2. 인증서 설치 — HTTPS 복호화를 위해 루트 CA를 시스템에 신뢰 등록합니다 (최초 1회, 관리자 암호 필요).
  3. 트래픽 연결
    • 맥/PC 전체: "시스템 프록시 ON" 토글
    • 특정 프로세스만: HTTP_PROXY=http://127.0.0.1:8888 HTTPS_PROXY=http://127.0.0.1:8888 <명령>
    • curl 테스트: curl -x http://127.0.0.1:8888 https://example.com/
  4. 재생 — 세션 옆 ▶ 버튼을 누르면 mock 서버가 127.0.0.1:8889에 뜹니다 (메서드 + 경로 매칭).
  5. 내보내기 — 세션 옆 내보내기 버튼으로 HAR / Markdown / curl / Postman / OpenAPI / k6로 저장합니다.

4. 자동 업데이트

설치형 앱은 실행 시 GitHub Releases를 확인해 새 버전이 있으면 자동으로 내려받아 다음 실행 때 적용합니다. 별도 설정은 필요 없습니다. (개발 모드에서는 동작하지 않습니다.)
